# REST Assured – Smallcase API Automation Framework

REST Assured is a java library and has been used here to create an end-to-end test framework for smallcase APIs

## Installation/ Pre-requisites
In order to run this project, following are required to be pre-installed
```bash
- Java (version 1.8 or higher)
- Maven 
```


## Reporting and Logs
Reports & logs will be generated in test-output as extentreports & Logs folder.
A default report gets generated by simply running the suite but few additional properties are added in the extent report such as tagging for each broker, writing request and response etc.
To use that, following things needs to be done while writing tests :
- Extend class base Test in each new testcase or existing testcase
- Implement interface Filter (RequestLoggingFilter class specifically) while creating a request
- Call writeRequestAndResponseInReport written in base test

## Sample
```java
@Test
public void sampleTest()
        {
        response = given().filter(new RequestLoggingFilter(printStream)).when().get();
        writeRequestAndResponseInReport(StringWriter, response, "tag")
        }
```

## Execution
To execute this project, clone the [repo](https://github.com/smallcase/restassured.git) and run command mvn clean test

```bash
git clone https://github.com/smallcase/restassured.git
mvn -Denv=${Environment} -Dsurefire.suiteXmlFiles=${xmlName} clean test  
${environment} = (staging || development || production)
${xmlName} = (config/end2end_flow.xml || config/main_runner.xml || config/order_flow_without_sip.xml || config/order_flow_with_sip.xml || config/page_navigation_flow.xml)
```


## YAML Configuration
The framework includes GitHub Actions workflows for automated testing:

### Available Workflows
- **dynamic-dispatcher.yml** - Manual test execution with suite selection
- **scheduled-jobs.yml** - Automated cron job execution (4 times daily)
- **dev-flow-POC.yml** - Development branch testing
- **prod.yml** - Production environment validation

### Scheduled Execution
```yaml
schedule:
  - cron: '30 9 * * 1-5'   # 3:00 PM IST - end2end_flow - production
  - cron: '30 10 * * 1-5'  # 3:30 PM IST - multi_asset_search - production
  - cron: '30 11 * * 1-5'  # 4:30 PM IST - stage_sam_flow - staging
  - cron: '30 14 * * 1-5'  # 7:00 PM IST - end2end_flow - staging
```

## Troubleshooting
- When running the tests if you get a NullPointerException in SureFire plugin, and there is no log from any of your code files in the error message, it might mean that something is breaking before you have initialized the Extent report. For example, if there is an API call in BeforeSuite and somehow that API call is failing, it might not show you the correct reason for failing tests, but just show a null pointer exception, if the api call is before initializing the ExtentSuite in BeforeSuite.
